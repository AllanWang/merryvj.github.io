Cartoon Portrait Generator: Live from your face!

View project
Fairly pointless. But here's what it does:
  • Webcam input of someone's face
  • Approximate their facial features into a mishmash of nose & eye doodles drawn by random strangers on the internet
  • Let someone download their masterpiece as an image
  • Perhaps they'll share it, frame it on their wall, update their social media profile pictures

Mildly horrifying, isn't it?

My first real fun with web development

I took ISS241: Web Project Design and Development in Spring 2019 and created this for my final project. I used NodeJS, Express.js, p5.js, plain old Javascript, and HTML/CSS. I also used a face-tracking API and Google's Quick Draw dataset. Because there was so much data from the Quick Draw dataset, I used Firebase to manage it.

But why build this?

Our final project was extremely open-ended. We were encouraged to use p5, a Javascript grapic library, to create an interactive experience.

As I was exploring ideas, I stumbled across Quick, Draw and immediately spent the rest of class playing with it.



I was inspired by projects people had built using the data: machine learning models, generating story scenes from spoken sentences, and even a smart "camera" that recognized objects in photos and turned them into a cartoon! So cool. So many possibilities. And if I could use Tensorflow? Woah!

But with a creeping deadline, I decided to build something more manageable. I remember playing with photo-to-portrait tools as a kid – could I make something similar? But of course, the portraits in my project would be much more abstract. A weird form of "art", perhaps?

The technical challenges

I had no idea how to get the raw, vector Quick Draw data working with p5. It took an immense amount of Googling and looking at the code of other remotely related projects to get a sense of where to begin–or if it would work at all.

The server stuff! The data! No more nice little static HTML pages. Figuring out how to set up a database to host all the vector drawing data, retrieve information from said database, and then updating it all live in my app.

Generated faces ended up being...scarier...than I expected. Since I could only use nose and eye drawings from Quick Draw, for rendering the rest of the facial features (mouth, lips, eyebrows, face shape), I had to work with the face tracking API to write something to approximate those facial features from scratch.

This kick-started my journey to learning how to build things

It's a whole new world out there! I want to keep playing around with web development to see what's possible – I just want to learn how to create cool (and hopefully useful) things. Knowing full-stack development is so powerful. My next steps are to learn more about the back-end, but also more front-end frameworks like React or Vue.
